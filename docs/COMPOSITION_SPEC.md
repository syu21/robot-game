# 編成・強化・進化仕様

最終更新日: 2026-04-02

## 1. 範囲
- 所持パーツ一覧: `/parts`
- ロボ編成: `/build`, `/build/confirm`
- パーツ強化: `/parts/strengthen`（`/parts/fuse`互換）
- 進化合成: `/parts/evolve`

## 2. 所持パーツ一覧
### 2.1 表示方針
- 装備中と所持中を同じ一覧で比較できる
- 在庫上限は `今すぐ使える所持枠`（`part_instances.status='inventory'`）にのみ適用する
- 上限超過で入手した個体は `overflow` 扱いで保管し、報酬は消失させない
- 前面に出す情報は `画像 / パーツ名 / 部位 / レアリティ / +値 / 6ステ / 装備中かどうか`
- `hp / atk / def / spd / acc / cri` などの内部キーはUIへ出さない
- 旧在庫などの内部都合ラベルは出さず、必要なときだけ `保管パーツ` として別扱いにする

### 2.2 比較情報
- 6ステはカード内グリッドで常時表示する
- `総合値` と `注目2能力` を補助表示する
- 装備中個体はバッジで明示し、`見比べる` 選択には含められる
- `破棄` は所持中個体だけを対象にし、装備中は誤って消えないようにする
- 保管中個体は別セクションで閲覧でき、選択して `所持へ戻す` 操作ができる
- ただし、強化・進化・編成の候補には混ぜない

### 2.3 絞り込み
- 部位フィルターを持つ
  - `すべて / 頭 / 右腕 / 左腕 / 脚`
- クエリパラメータ `part_type` で保持する
- ページ送りと併用しても条件が落ちない

### 2.4 在庫表示
- ホームでは `所持 X/Y | 保管 Z` で表示する
- `保管 Z` は `overflow` 個体と旧 `user_parts_inventory` を合算する
- 保管中の個体パーツは `/parts` から選択して所持へ戻せる
- 所持一覧の主操作は `見比べる / 破棄` に分け、`整理` のような曖昧な文言は使わない
- 旧 `user_parts_inventory` の破棄UIは従来どおり維持する

## 3. ロボ編成
### 3.1 必須・任意
- 必須: `HEAD`, `RIGHT_ARM`, `LEFT_ARM`, `LEGS`
- 任意: `DECORATION`

### 3.2 比較表示
- 編成候補カードでも `総合値 / 注目能力 / 6ステ` を見ながら選べる
- プレビュー比較（組み立て中 vs 現装備）は維持する

### 3.3 合成画像
- 画像サイズ: 128x128 PNG
- レイヤー順: `LEGS -> HEAD -> RIGHT_ARM -> LEFT_ARM -> DECORATION`
- オフセット: `robot_parts.offset_x/y` を使用

### 3.4 保存
- `robot_instances` + `robot_instance_parts` に保存
- 保存枠超過時は保存ブロック

### 3.5 小型ユーザー機体アイコン
- `robot_instances.icon_32_path` に 32x32 の小型機体画像を保持する
- 生成タイミングは `/build/confirm` の編成確定時を正とし、同時に `users.active_robot_id` を更新する
- 未生成時だけ `composed_image_path` からオンデマンド再生成する
- 切り抜きは `HEAD` が見やすい上寄せの正方形を優先し、機体全体を縮小しすぎない
- `DECORATION` を積んでいる場合は、本体を隠さない範囲で小さく残す
- ユーザー表示では、この `icon_32_path` を `主役の小ロボ画像` として使う
- 補助アバターは `Google画像 -> 手動画像 -> seed生成アイコン` の順で選ぶ
- 機体未所持時だけ `主役の小ロボ画像` は `初期ノーマルロボ顔` の既定画像へフォールバックする
- 用途:
  - `/ranking` の user系ランキング
  - `/home` のヘッダー、`今週のランキング`、`今週のMVP`
  - `/world` の MVP 表示
  - `/records` のユーザー表示
  - `/comms/world`, `/comms/rooms` のユーザー表示

## 4. DECOR表示方針
- DECORはバッジ的に小型表示（本体中心を隠さない）
- 所有DECORのみ選択可
- `なし` は常時選択可

## 5. パーツ強化
### 5.1 導線
- 正規: `/parts/strengthen`
- 互換: `/parts/fuse`（同処理）

### 5.2 仕様
- ベース1 + 素材2
- 成功率100%
- 上昇量は常に +1固定
- 対象条件は「同名 + 同レアリティ」（+値違い可）
- 強化候補は「成立可能なグループ」のみ表示
- 素材自動選択時は安全制約あり（装備中素材消費禁止など）

### 5.3 比較表示
- ベース候補は個体単位で選ぶ
- 画面内で `強化前 -> 強化後` の差分を6ステで見せる
- 同時に、素材として消える2個も確認できる
- 装備中ベースを使う場合は、素材が所持中から出ることを文言で明示する
- 保管中個体は強化候補に混ぜず、保管のため成立していない組み合わせがある場合は `/parts` へ戻す案内を出す
- 失敗時は `失敗` 表示と理由文を結果面に残し、無反応に見せない

### 5.4 監査
- `audit.fuse`
- payloadに成功可否、消費ID、増分などを保持

## 6. 進化合成
### 6.1 仕様
- Nパーツ + 進化コア1個 -> 対応するRパーツ
- 進化先Rマスタ未登録ならエラー復帰

### 6.2 引き継ぎ
- `plus` 維持
- `w_hp..w_cri` 維持

### 6.3 比較表示
- 候補カード上で `進化前 -> 進化後` を見比べられる
- 必須表示は `パーツ名 / 部位 / レアリティ / +値 / 6ステ`
- `強化値と個体性能はそのまま引き継がれます` を短く補助表示する

### 6.4 監査
- `audit.part.evolve`

## 7. 表示名

  2. 自動生成名
