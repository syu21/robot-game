# UGC導入ロードマップ（ロボらぼ）

最終更新日: 2026-03-26

## 1. 基本方針
- 戦力差を作るUGCは段階導入しない
- 構造パーツは公式供給を主軸に維持
- UGCは DECOR / 表示価値 から始める
- 監査可能性と不正耐性を先に整える
- 現フェーズでは `周回快感` と `間接競争` を優先する
- UGC は PvP の前に `見栄 / 誇示 / 展示 / 語り` を強化するための要素として扱う
- 中長期方針の正本は `docs/GAME_DIRECTION.md`

## 2. 現在地（実装済み）
### 2.1 DECOR基盤
- `robot_decor_assets` 管理
- `user_decor_inventory` 所有管理
- `/admin/decor` で登録・有効化
- ボス撃破でDECOR付与（重複抑止）

### 2.2 関連導線
- `/build` でDECOR選択
- battle結果でDECOR獲得反映
- 監査で付与経路追跡
- `/showcase`, `/world`, `/records` で機体の見栄えと話題化導線を受け止める面がある

### 2.3 表示名統制
- `robot_parts.display_name_ja` を正本表示名に採用
- 未設定時は生成フォールバック

## 3. 直近フェーズ（Phase A）
### 3.1 DECOR投稿（承認制）
- 投稿は下書き保存
- 公開は管理承認後
- 画像要件:
  - PNG
  - 透過必須
  - サイズ規定

### 3.2 モデレーション
- NGワードチェック
- 著作権侵害対応導線
- 無効化フラグで即時停止

## 4. 中期フェーズ（Phase B）
- テーマ別DECORイベント
- 展示連携（ロボ展示で見栄価値拡張）
- 称号連携（戦力効果なし）
- 世界ログ連携（話題化 / 語り化）
- ランキング連携（間接競争の可視化）

## 5. 将来フェーズ（Phase C: パーツUGC検討）
- パーツUGCは即公開しない
- 運営側で `display_name_ja` を付与/統制
- 接続規格・見た目規格・監査を整備後に限定解放

## 6. 非目標（現時点）
- ユーザー投稿パーツの即戦力化
- 課金で有利になる能力差
- 監査を通らない即時配布
- 早期 PvP 前提の戦力UGC

## 7. 成果指標
- DECOR投稿数
- 承認率
- 通報率
- 展示導線の滞在時間
