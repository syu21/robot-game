# 戦闘仕様（現行実装）

最終更新日: 2026-04-04

このドキュメントは、現在のロボらぼに実装されている戦闘仕様の正本です。  
設計意図や理想形は [docs/BATTLE_DESIGN_SHEET.md](/Users/ootanisyuhei/Desktop/キカビルド/robot-game/docs/BATTLE_DESIGN_SHEET.md) を参照し、このファイルでは「今どう動いているか」を優先して記述します。

## 1. 戦闘の入口

現在の戦闘入口は2系統あります。

- 主戦闘: `POST /explore`
  - ロボらぼの本流です
  - 出撃 -> 自動戦闘 -> 戦利品表示 の流れで使います
- 旧戦闘: `/battle`
  - クリック式の簡易戦闘です
  - 残っていますが、メイン体験は `/explore` 側です

以下では、まず主戦闘である `/explore` の仕様を書き、その後に `/battle` の現状を補足します。

## 2. `/explore` の前提条件

`POST /explore` は次の順にチェックします。

1. ログイン済み
2. `area_key` が有効
3. 対象エリアが公開中
4. 層解放状態を満たしている
5. アクティブ機体が存在する
6. 出撃CTを満たしている

CT は共通ヘルパーで判定します。

- 一般ユーザー: 40秒
- 登録直後の新規ブースト期間: 20秒
- `explore_boost_14d` 購入中: 20秒
- 管理者: 0秒

重要なのは、UI の残り時間表示とサーバ側判定を同じロジックで揃えていることです。

## 3. 戦闘に入る前に確定する値

戦闘開始時点で、プレイヤー側は以下を確定します。

- アクティブ機体の最終ステータス
  - `hp`
  - `atk`
  - `def`
  - `spd`
  - `acc`
  - `cri`
- 機体アーキタイプ
- ロボスタイル
- 戦闘モード
  - `normal`
  - `berserk`
- ビルド種別
  - `STABLE`
  - `BURST`
  - `BERSERK`
- ステージ補正
- 週間環境補正

### 3.1 ビルド種別

現在のビルド種別は次の扱いです。

- `STABLE`
  - 4部位の属性がすべて同じとき
- `BURST`
  - 上記以外の通常ビルド
- `BERSERK`
  - 機体の `combat_mode = berserk` のとき

### 3.2 ビルドによる補正

- `STABLE`
  - ダメージ乱数: `0.95 - 1.05`
  - 会心倍率補正: なし
- `BURST`
  - ダメージ乱数: `0.80 - 1.25`
  - 会心倍率: 基本会心倍率に `+15%`
- `BERSERK`
  - 初期最大HPを `-15%`
  - 失ったHPに応じて攻撃補正が上がる
  - 攻撃補正は `missing_hp_ratio * 0.60`、最大 `+30%`

## 4. 1回の出撃で何戦するか

基本は `1出撃 = 1戦` です。  
ただし週間環境 `暴走` のときは、25% で `2連戦` になります。

連戦時の重要仕様:

- プレイヤーHPは次戦へ持ち越します
- 敵HPは各戦でリセットされます
- 2戦目に入る前に区切りログが入ります
- 連戦ボーナスとしてコイン `+1` が1回だけ加算されます

## 5. 1戦の基本ルール

- ターン上限: `8`
- 両者生存のまま8ターン到達: `timeout`
- 先手は `spd` 比較
- 同値時はプレイヤー先手

1ターンの流れは単純です。

1. 先手側が攻撃
2. 相手が生きていれば後手側が攻撃
3. どちらかのHPが0なら終了

## 6. 攻撃計算

攻撃計算の本体は `services/simulate_balance.py` の `resolve_attack()` を使っています。

### 6.1 命中率

命中率は次の式です。

```text
hit_chance = clamp(0.75 + (att_acc - def_acc) * 0.01 + hit_bonus, 0.60, 0.95)
```

補足:

- 基本命中率は 75%
- 命中差 1 ごとに ±1%
- `sniper` アーキタイプは `+3%`
- 下限 60%、上限 95%
- `force_hit=True` のときだけ MISS を無視

### 6.2 ダメージ

基本ダメージは次です。

```text
base_damage = max(1, (ATK - DEF * 0.5) + randint(-1, 1))
```

その後、ビルドに応じた乱数レンジを掛けます。

- `STABLE / BERSERK`: `0.95 - 1.05`
- `BURST`: `0.80 - 1.25`

### 6.3 会心

会心率は次です。

```text
crit_chance = clamp(CRI * 0.01, 0.01, 0.25)
```

- 下限 1%
- 上限 25%
- 基本会心倍率は `1.5`
- ただし敵やビルドで倍率が変わることがあります

### 6.4 アーキタイプ補正

現在の `resolve_attack()` で使っている直接補正は次です。

- 攻撃側 `sniper`
  - 命中率 `+3%`
- 攻撃側 `swift` かつ先手
  - ダメージ `+10%`
- 防御側 `fortress`
  - 被ダメージ `-10%`

## 7. `/explore` 戦闘での追加補正

`/explore` は `resolve_attack()` の前後で、探索専用の補正も加えています。

### 7.1 tier1救済

通常の tier1 敵だけ、連続MISS救済があります。

- 対象: 非ボスかつ `tier == 1`
- プレイヤーが 2回以上連続MISS
- 次回攻撃を `force_hit=True` にする

つまり、序盤だけ「当たらなすぎて何も起きない」を少し減らしています。

### 7.2 敵 trait

敵 trait は現在次のように働きます。

- `heavy`
  - プレイヤーの通したダメージを `15%` 軽減
- `fast`
  - プレイヤーの命中攻撃が 12% で回避に化ける
- `berserk`
  - 敵HPが半分以下になると敵ATK `+20%`
- `unstable`
  - 敵攻撃後 35% で反動発生
  - 反動量は `enemy_atk * 12%` 相当、最低1

### 7.3 背水

`combat_mode = berserk` の機体は、プレイヤー攻撃時に毎ターン再計算で攻撃補正が乗ります。

- HPが減るほど攻撃が上がる
- ただし最大 `+30%`

### 7.4 ステージ補正

エリアごとにステージ補正を持つ場合があります。

プレイヤー側に掛かる可能性がある値:

- `atk`
- `def`
- `acc`

敵側に掛かる可能性がある値:

- `atk`
- `def`
- `spd`
- `acc`
- `cri`
- `hp`

管理者はステージ補正を無視します。

## 8. ボス戦

### 8.1 基本出現

エリアボス抽選は、対象エリアで内部管理しています。

- 基本値は多くの層で `0.5%`
- `layer_4_final` と `layer_5_final` は `100%`

### 8.2 pity

第1〜第3層だけ、内部 pity を持ちます。

- `layer_1`
  - soft pity 開始: 45回
  - 強制到達: 90回
  - soft pity 最大率: 3.5%
- `layer_2`
  - soft pity 開始: 70回
  - 強制到達: 120回
  - soft pity 最大率: 4.0%
- `layer_3`
  - soft pity 開始: 85回
  - 強制到達: 140回
  - soft pity 最大率: 4.5%

第4層以降は pity を実質無効にして、ボスのレア感を維持しています。

### 8.3 ボス警報

通常探索でボスに当選した時点では、その場で即戦闘には入らず `ボス警報` を付けます。

- 挑戦回数: `3回`
- 有効時間: `45分`
- `boss_enter=1` 付きで出撃すると警報を消費してボス戦に入ります
- 回数が 0 になると警報は消えます

管理者は、警報なしでもボス戦へ直接入れます。

## 9. 勝敗と報酬

### 9.1 勝利時

通常戦勝利では以下が発生します。

- コイン加算
- 経験値 `+2`
- パーツドロップ抽選
- 進化コア抽選
- 敵図鑑更新
- 連勝管理
- 週間環境ボーナス反映
- 陣営ボーナス反映

### 9.2 敗北時

- コインは基本なし
- 戦利品なし
- その時点で出撃終了

### 9.3 進化コア

進化コアは3系統あります。

- 通常ドロップ
- 勝利保証ゲージ満了
- NPCボス撃破報酬

### 9.4 ボス報酬

- 固定ボス / エリアボス
  - DECOR 報酬中心
- NPCボス
  - 進化コア報酬

### 9.5 連勝

同じ地点で `3連勝` するとコイン `+1` が入ります。  
別エリアへ移ると、その地点以外の連勝はリセットされます。

## 10. 戦利品表示

`/explore` 後は `battle.html` に結果を返します。  
前面で見せるのは最小限です。

- 勝敗
- コイン
- パーツドロップ
- 進化コア
- ボス報酬
- 次の行動

`次の行動` は次を出します。

- もう一度出撃
- 入手したパーツを見る
- 基地へ戻る

CT中は `もう一度出撃（あとN秒）` に変わり、0秒で自動活性化します。

## 11. 戦闘演出 `battle-cinematic-v1`

現在の `/explore` 戦闘は、結果画面の前に `battle-cinematic-v1` を表示します。  
これは連続動画風ではなく、`1ターン = 1カード` の半コマ送り演出です。

### 11.1 基本仕様

- サーバー側で `turn_logs` から演出専用 payload を組み立てる
- payload は `summary["battle_cinematic"]` に載せ、互換用に `summary["battle_replay"]` にも同じものを入れる
- フロントは `templates/_battle_cinematic_v1.html` と `static/battle_cinematic_v1.js` で自動再生する
- 既存の勝敗・ダメージ・会心・ドロップ判定はそのまま使い、演出専用 payload だけを追加する

### 11.2 表示モード

表示モードは 3 種あります。

- `標準`
  - 初期値
  - 1ターンごとにカードと戦況文を見せる
- `高速`
  - 同じ turn card を短めに再生する
- `即結果`
  - 演出を飛ばして結果セクションを即表示する

`標準 / 高速` は `localStorage` に保存します。  
`即結果` はその場だけの一時操作です。

### 11.3 1ターンで見せる要素

各ターンカードでは次を出します。

- `TURN n`
- 行動者
- 行動ラベル
- 結果ラベル
- 補助値ラベル
  - `MISS`
  - `BLOCK`
  - `-18`
  など
- 機構状態ラベル
- 戦況要約

戦況要約はダメージログの言い換えではなく、意味の要約です。

### 11.4 機構状態

v1 では部位破壊を使わず、次の `機構状態` を演出用ラベルとして出します。

- `装甲亀裂`
- `センサー異常`
- `駆動低下`
- `武装不安定`
- `出力暴走`
- `体勢維持`

状態は戦闘ロジックの本体には影響させず、既存のダメージ量・会心・MISS・敵 trait・残HP比率などから演出専用に推定します。

### 11.5 画面上の見せ方

- 左に自機、右に敵機
- 小さめ HP バー
- 状態バッジ
- 中央に弾道 / 着弾 / フラッシュ / 火花
- 下部にターンカード
- 最後に `今回の勝ち筋 / 崩れ筋`

勝利時の総括例:

- `爆発力で押し切った`
- `命中安定で崩した`
- `体勢維持で受け切った`
- `装甲差で競り勝った`

敗北時の総括例:

- `火力差を受け切れなかった`
- `命中差を返せなかった`
- `先手を失って崩れた`
- `崩れから立て直せなかった`

### 11.6 公開制御とフォールバック

公開は release gate で管理します。

- フラグ: `battle_short_replay`
- 非公開中:
  - 管理者のみ表示
  - 一般ユーザーには非表示

JS が失敗した場合は、battle-cinematic-v1 を開かず、そのまま既存の結果表示へ落とします。  
白画面にはしません。

## 12. ログと監査

### 12.1 turn_logs

`/explore` の戦闘詳細は `turn_logs` に積みます。  
主に次を持ちます。

- ターン番号
- 戦闘番号
- 各種導入行
- プレイヤー行動 / 敵行動
- 戦闘前後HP
- 与ダメ / 被ダメ
- 会心有無
- MISS / 0ダメ補助説明
- trait 発動行
- 救済行
- 背水行
- 最終結果行

既存の互換制約として、`turn_logs` の意味は変えない前提です。

### 12.2 audit

主な監査イベント:

- `audit.explore.start`

## 13. 旧 `/battle` の現状

`/battle` は現在も残っていますが、主戦闘ではありません。

現状仕様:

- 敵HPは固定 `5`
- 敵名は簡易ランダム
- プレイヤー操作:
  - 攻撃
  - 防御
  - 逃走
- 一般ユーザーは行動CT `20秒`
- 管理者はCTなし
- 攻撃ダメージは `click_power` ベースの簡易乱数
- 敵撃破で:
  - コイン `+1`
  - 勝利数 `+1`
  - パーツドロップ抽選
  - 低確率でロボ入手

このルートは legacy 的に残っているため、今後の主仕様を議論するときは `/explore` を基準にしてください。

## 14. まとめ

今のロボらぼ戦闘は、次の性格を持っています。

- 本体は `/explore` の自動戦闘
- 1戦は最大8ターン
- 先手は `spd`
- 命中は `acc` 差ベース
- 会心は `cri` ベース
- tier1だけ救済あり
- 敵 trait とビルド差で挙動が少し変わる
- ボスは警報制で挑戦権を消費
- 結果画面前に短いショート演出を差し込む
- 戦利品表示が主役

つまり、長い読み合いの戦闘ではなく、  
`短時間で回れるが、型差と成長差は残る戦闘` が現行仕様です。
