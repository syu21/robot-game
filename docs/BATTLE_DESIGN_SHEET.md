# 戦闘・経済 設計シート

最終更新日: 2026-03-28

## 1. 設計目的
- 短時間で1周できるテンポ
- 初見でも理解しやすい結果画面
- 研究余地は残すが前面情報は最小化

## 2. 戦闘テンポ
- 基本: 1出撃1戦
- 例外: 特定環境で連戦あり
- 1戦の上限: 8ターン
- 目標ターン帯: 3〜5ターン
- 第4層は上限を変えず、通常敵から `layer_3` 固定ボス級を超える強度にする
  - チュートリアル終了後の本番帯として、全体火力と耐久を一段引き上げる
  - そのうえで敵の役割差で実質ターン数を分岐させる
  - forge: やや長期戦
  - haze: MISSで伸びやすい
  - burst: 短期高刺激
- 第5層も上限は変えず、4層よりさらに高圧な終盤帯として勝ち方の軸を複数化する
  - labyrinth: 安定して抜けるか
  - pinnacle: 最速で抜けるか
  - omega: 何を捨てて何を残すか

## 3. 勝敗体験
- 勝利時: 戦利品カードを最短で確認
- 敗北時: 理由が分かるログを残し再挑戦導線を明示
- 重要: 通常戦は静か、ボス撃破時のみ演出を強める
- 第4層の役割は「背水一強をナーフではなく相性分化で崩すこと」
- 耐久 / 命中 / 爆発 / 安定にそれぞれ勝ち場所を作る

## 4. 戦利品UI方針
前面表示は以下のみ:
- 獲得コイン
- ドロップ結果（なしの場合は `戦利品なし`）
- 進化コア保証やボス報酬の短い結果表示

前面から外す情報:
- 内部補正係数
- 出現率/ドロップ率の数式寄り表示
- 構成メモ等の分析項目

## 5. ボス戦方針
- 出現率 0.5% を基準
- 固定ボスは層進行に紐づけ
- 固定ボス/エリアボス報酬は DECOR 中心
- NPCボスは進化コア報酬あり
- SYSTEM投稿はボス撃破時のみ
- 第4層は 3サブエリア + 最終試験で型の理解を学ばせる
- 第5層は「型を誇る層」として、週環境 / 記録 / 展示へ接続する前提で設計する
- 第5層の価値は単純な高難度ではなく、安定記録と最速記録の両方を立てられること

## 6. 成長導線との接続
- 強化: `ベース1 + 素材2 -> +1固定`
- 進化: N + 進化コアでR化
- 保証: 勝利の積み上げで進化コア保証進捗が前進
- 編成: 4部位 + DECORで役割調整

## 7. ログ方針
- MISS/0ダメは抽象カテゴリで説明
- 詳細数式は常時表示しない（必要時のみ詳細へ）
- 監査は `audit.*` で別保存

## 8. 安定性要件
- CT表示とサーバ判定は一致
- JSエラー時でも白画面を避ける
- overlay演出はページ跨ぎで残留させない
