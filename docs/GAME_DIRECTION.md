# ロボらぼ 運営思想メモ

最終更新日: 2026-03-26

この文書は、ロボらぼの中長期方針を忘れないための運営用メモです。
実装提案、改善案、イベント企画、収益化案を考えるときは、この文書を前提に判断してください。

具体的な実装順は `docs/IMPLEMENTATION_ROADMAP.md` を参照してください。

## 1. 一文で言うと

ロボらぼは、探索場所による育成差からロボの型・思想・役割を自然発生させ、その思想が世界ログ・ランキング・陣営戦などの間接競争で価値を持ち、十分に文化が育った後に最終解放として PvP に接続するゲームである。

## 2. 現在地

現在のロボらぼは `チュートリアルフェーズ` にある。

今入っている要素:
- 探索
- 強化
- 進化
- ボス
- DECOR 取得

これらはすべて、最終ゲーム体験の完成形ではなく `基礎体験の提供段階` とみなす。

つまり、現時点で最も重要なのは:
- 短時間周回が気持ちいいこと
- 何度も触りたくなること
- 育成差が自然に出始めること
- 他人の進行が見えて世界が動いているように感じられること

## 3. 先に忘れてはいけない結論

`PvP は入口ではなくゴールである。`

PvP は早期実装しない。
ロボらぼにおける PvP は「最初に置く主戦場」ではなく、「人口・ビルド多様性・競争文化が育った後に解放する最終コンテンツ」と位置付ける。

## 4. なぜ PvP を急がないのか

### 4-1. 人数が足りない段階で入れるとゲームが死ぬ

人口が十分でない段階で PvP を入れると、次が起きやすい:
- マッチしない
- 格差マッチになる
- 初心者狩りが起きる
- 負け体験が早すぎて離脱を招く

ロボらぼは、まず周回と育成で中毒性を作るべきであり、早期の対人敗北体験を主役にしてはいけない。

### 4-2. ビルド差が浅い段階で PvP を入れると最適解が一つになる

PvP が面白くなる前提は:
- 複数の強い型が存在すること
- プレイヤーごとの思想差が出ること
- 正解が一つに収束しにくいこと

今後この条件を作るために、探索場所差・育成差・思想差を先に育てる必要がある。

### 4-3. ロボらぼの PvP は勝敗装置ではなく思想の発表会であるべき

理想の PvP は、単なる勝ち負けの場ではない。

ロボらぼにおける PvP は:
- 自分の型を見せる場
- 積み上げた思想を発表する場
- 他人の構築を見て研究したくなる場

であるべきで、そこに到達する前に解放しても本来の価値が出ない。

## 5. 中長期進行構造

ロボらぼの正しい進化ルートは次の順番で考える。

### 段階1. 探索場所による育成差

探索場所ごとに、努力値のような育成傾向差が生まれる。

例:
- 廃工場系: 攻撃寄り
- 霧区系: 回避・命中差寄り
- 要塞系: 防御寄り

ここで重要なのは、単に数字が上がることではなく `どこを回ったかでロボの育ち方が変わる` ことである。

### 段階2. 育成差から型・思想・役割が生まれる

探索差が積み上がると、ロボに次のような個性が生まれる。

- 耐久型
- 速攻型
- 命中重視型
- 爆発型
- ボス特化型
- 周回安定型

ここでの価値は「強さそのもの」ではなく、`どう育てたか` と `なぜその構成なのか` が語れることにある。

### 段階3. その思想が間接競争で価値を持つ

型や思想は、まず PvP ではなく次のような場所で価値を持たせる。

- 世界ログ
- ランキング
- ホームのランキング表示
- 展示
- 陣営戦
- MVP 表示
- 最速記録
- 特定思想ランキング

これは `擬似対人競争` である。

直接殴り合わなくても、
- 誰が早いか
- 誰が変な型を作ったか
- 誰の思想が話題か

が見えるだけで、十分に競争と文化は発生する。

### 段階4. 最後に PvP を解放する

人口、ビルド多様性、研究文化、競争文化が育ってから、最終段階として PvP を解放する。

このとき初めて:
- 型
- 誇り
- 名声
- 研究成果

がぶつかり合う。

## 6. ロボらぼは「努力値 -> PvP」ではない

短く言い直すと、ロボらぼの正しい変換はこれである。

- 誤: `努力値 -> PvP`
- 正: `努力値 -> 周回思想 -> 世界競争 -> PvP`

この順番を崩さないこと。

## 7. 当面の運営目標

しばらくの間、ロボらぼの運営目標は次の4つに固定する。

### 7-1. 短時間周回の快感最大化

- 毎回同じ入力をさせない
- 再出撃しやすい
- テンポが良い
- 勝ち体験が途切れにくい
- 何周もしたくなる

### 7-2. ビルド差の自然発生

- 探索先や育成先で差が出る
- 同じレア度でも思想差が出る
- 一本道の最適解に寄りすぎない
- プレイヤーごとに育成経路が分かれる

### 7-3. 世界競争ログの強化

- 世界ログで出来事が見える
- ランキングがホームから自然に目に入る
- ボス撃破、進化成功、周回量が可視化される
- 「世界が動いている」感じを出す

### 7-4. 間接競争の可視化

- 他人が進んでいるのが見える
- 最速・最多・最高が見える
- 陣営ごとの空気が見える
- 展示や記録から真似・研究・語りが生まれる
- `アイコン+小ロボ` や機体画像で「人がいる感じ」を出す

## 8. 収益化の前提

収益化は `戦力販売` をしない。

軸にするのは次である。

- 見栄
- コレクション
- 便利性
- 誇示
- 参加感

具体的に相性が良い方向:
- 見た目
- 称号
- DECOR
- 展示強化
- 参加バッジ
- 履歴/記録の演出
- 周回を気持ちよくする軽い QoL

相性が悪い方向:
- 露骨な戦力販売
- 直接勝率を売る施策
- 新規が追いつけなくなる有料優位

## 9. これからの提案判断基準

今後の新機能は、必ず次の3条件で判断する。

### 条件1. 周回が気持ちよくなるか

確認観点:
- 入力ストレスが減るか
- 何周もしたくなるか
- テンポが良くなるか
- 成果の気持ちよさが増すか

### 条件2. 競争が自然発生するか

確認観点:
- 他人と比べたくなるか
- 誰かの進行が見えるか
- 自然にランキングや記録が生まれるか
- 陣営・展示・世界ログへ繋がるか
- 画面に他プレイヤーの存在感が出るか

### 条件3. プレイヤーの語りが生まれるか

確認観点:
- スクショを貼りたくなるか
- X や Discord で話題になるか
- 自分のロボ思想を語れるか
- 他人の構築を見て真似や研究が生まれるか

## 10. 良い提案の例

- 前回の出撃先を記憶する
- ホームから再出撃しやすくする
- ボス撃破を世界ログに出す
- 進化成功を世界ログに出す
- 今週ランキングをホームから見せる
- 型や思想が見える展示を増やす
- 陣営ごとの進行差を見せる
- 最速・最多・最硬・最大火力などの記録文化を作る

## 11. 悪い提案の例

- 人口が少ない段階で PvP を急ぐ
- 説明を長文で足してホームを重くする
- 一本道の最適解をさらに強める
- 周回のたびに同じフォーム入力を要求する
- 内部監査ログをそのまま公開してノイズを増やす
- 課金で明確な戦闘優位を売る

## 12. 今後の優先順メモ

直近は次の順で考える。

1. 周回ストレスの除去
2. 世界ログの強化
3. ランキングと展示の強化
4. 型・思想の可視化
5. 陣営による擬似対人競争
6. PvP は最後

## 14. 最重要フレーズ

忘れそうになったらこの3つを見返すこと。

- `PvP は入口ではなくゴール`
- `努力値 -> 周回思想 -> 世界競争 -> PvP`
- `新機能は「周回快感」「自然競争」「語りの発生」の3条件で判断する`
