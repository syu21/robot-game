# ロボらぼ 実装優先ロードマップ

最終更新日: 2026-04-02

この文書は、ロボらぼの実装優先順位を忘れないための正本メモです。
`docs/GAME_DIRECTION.md` に書かれた運営思想を、具体的な実装順に落としたものとして扱います。

新機能、改善案、UI調整、運用施策を考えるときは、
「今どの Phase を進めているか」
「その実装が Phase の目的に合っているか」
を必ず確認してください。

## 1. この文書の前提

ロボらぼは現在 `チュートリアルフェーズ` にある。

今の探索・強化・進化・ボス・DECOR取得は、
完成したゲーム本編ではなく `基礎体験の提供段階` である。

現時点の診断を短く言うと、
`ゲームはできているが、世界の熱量がまだ足りない` 段階にある。

第3層までの導線とコアループは成立しつつある一方で、
プレイヤー人口が少ないため `世界が動いている感じ` が弱く、
無人感が体験価値を削っている。

当面の優先は次の通り:

- β版として `公開入口(/ -> /register) -> 登録 -> ホーム -> 初戦闘 -> 戦利品 -> 再出撃/育成` の入口体験を磨く
- 周回快感を上げる
- 型 / 思想の自然発生を作る
- 世界ログ / ランキング / 展示 / 陣営戦で間接競争を育てる
- プレイヤー同士の研究・語り・記録文化を作る
- 少人数でも `人がいる / 世界が動いている` 感じを成立させる

`PvP は入口ではなくゴール` とする。
早期に PvP を主役にしない。

## 2. 判断基準

今後の実装は、必ず次の3条件で判断する。

### 2-1. 周回が気持ちよくなるか

確認観点:

- 毎周回の入力ストレスが減るか
- 前進感が増えるか
- 勝ち体験や報酬感が気持ちよくなるか
- もう1周したくなるか

### 2-2. 競争が自然発生するか

確認観点:

- 他人との比較軸が自然に生まれるか
- ランキング、記録、展示、陣営戦に繋がるか
- プレイヤーごとの差が可視化されるか
- 間接競争を強めるか

### 2-3. プレイヤーの語りが生まれるか

確認観点:

- スクショを貼りたくなるか
- X / Discord で話題にしやすいか
- 自分の育成思想を語れるか
- 他人のロボを見て真似や研究が生まれるか

## 3. 固定制約

今後の実装では、次を壊さないこと。

- 8ターン戦闘を維持する
- `audit.*` 互換を維持する
- `turn_logs` 互換を維持する
- `SYSTEM` 投稿の既存方針を壊さない
- 実装変更時は `docs` 更新と `tests` 更新を必須とする
- 公開は `一気に見せる` のではなく、`管理者確認 -> 段階公開` を優先する

## 4. 実装優先順

### β版入口改善

短期的な最優先は、新機能追加ではなく `入口〜初戦闘` を完成させること。
現行の公開入口はまだ `/register` 中心で、`/` の専用LPは準備段階とみなす。

優先順:

1. 公開入口（現状 `/register`、将来的に `/`）で `何のゲームか / 世界が動いているか / 今始めていいか` を3秒で伝える
2. 登録を軽くし、Google登録を含めて登録直後にホームからそのまま第1層へ出撃できるようにする
3. ホームの `最初のミッション` と `Next Action` を、最初の1勝へ直結する文言に寄せる
4. 戦利品結果面で `もう一度出撃 / 入手したパーツを見る / ホームへ戻る` を短く出し、押しやすくする
5. 支援導線は控えめに1本だけ見せ、強さ販売に見えないようにする

## Phase 3.5: 世界創造フェーズ

目的:

- 無人感を消し、世界が常に動いて見える状態を作る
- 少人数でも「遊ばれている世界」に見える基盤を作る
- 新規流入時に、静かなゲームではなく `動いている世界` として返す

現在地メモ:

- 2026-03-29 時点では、第3層までの体験は成立しつつある
- 問題は `ゲーム性不足` より `世界の活動量不足` に寄っている
- 当面は新モード追加より `視覚 / ログ / 用事` を優先する
- 今やるべきことは `ロボを動かす / 人がいる感を作る / 今日の用事を作る`
- 2026-04-02 `通信 / 世界戦況 / 記録庫 / ランキング` で recent activity と presence 表示を先行反映した

実装原則:

- 重い GIF ではなく、CSS アニメーションや 2〜4 コマの軽量スプライトで見せる
- `人がいる感` は、実ユーザー数の偽装ではなく `実活動ログ + 世界設定に沿った存在演出` で作る
- BOT や補助アカウントを使う場合は、実プレイヤーと誤認させるためではなく、世界内の `研究員 / 訓練機 / 展示機` として扱う
- リアルタイム PvP やマッチングは、この段階では主役にしない
- 新規プレイヤーが 10 秒で「何かが起きている」と理解できる画面を優先する

順序:

### 16. ロボの軽量アニメーション

狙い:

- 画面の静止感を消す
- 小型機体アイコンの存在感を増やし、世界が生きている印象を作る
- 滞在時間と再訪率を上げる

完成条件の例:

- `ホーム / 今週のMVP / ランキング / 世界ログ` の小型機体アイコンが軽く動く
- 常時は `ふわふわ / 呼吸 / まばたき` などの軽い動きに留める
- `出撃中 / 待機中 / 放置` で速度差をつけられる余地を持つ
- 重い画像演出や大量通信なしで描画できる

### 17. 活動ログと在線感の強化

狙い:

- 「人がいる感」を強制的に可視化する
- 世界ログと会議室を、単なる投稿置き場ではなく `世界の呼吸` に変える

完成条件の例:

- `ログイン / 出撃開始 / ボス撃破 / 進化成功` が世界ログに自然に流れる
- 会議室や通信で `最近5分で動いた人数` のような軽い存在表示ができる
- `今○人が活動中` のような表示は、実計測ベースで作る
- 活動量が少ない時間帯でも、世界設定に沿った補助ログで空白感を減らせる

### 18. 今日の用事を作る

狙い:

- 再訪理由を作る
- 「遊ぶ理由がない」を防ぎ、少人数でも日次の習慣を作る

完成条件の例:

- `今日の出撃 / 今日のボス / 今日の挑戦` が短く見える
- 1 回のログインで達成できる軽い用事から始める
- 報酬は強さを壊さず、通貨 / DECOR / 記録系で回す

### 19. 比べる文化の下準備

狙い:

- PvP を入れる前に、比べる文化だけを先に作る
- 展示・記録・挑戦の流れを強くする

完成条件の例:

- `今週の挑戦ロボ` のような全員共通の相手がある
- 展示ロボとの模擬戦や AI 戦の入口を検討できる
- `復讐` のような感情導線は、非同期形式で扱う
- リアルタイム PvP なしでも「比べたい」が生まれる

### 20. 流入向け体験投稿の型を作る

狙い:

- 集客を `更新報告` ではなく `体験投稿` に寄せる
- クリックしたくなる入口を増やす

完成条件の例:

- `10秒でロボを作れる`
- `このロボどう育てる？`
- `命中999が出た`

のような短い投稿テーマが運営内でテンプレ化されている

次段階メモ:

- 収益化は `強さ販売` ではなく `時間 / 見栄 / 参加` を軸に設計する
- `異常ロボ` は強い案だが、世界が動き始めてから投入する
- 非同期 PvP は有力候補だが、リアルタイム PvP より先に `展示 / AI / 今週の挑戦` を固める

## 5. 今やらないこと

現時点では、次を主戦場にしない。

- 早期 PvP
- リアルタイム PvP / マッチング
- 長文チュートリアルの大量追加
- ホームの説明カード増殖
- 一本道の最適解を強める実装
- 直接勝率を売る収益化
- 実人数を誤認させる人口演出
- 重い GIF 前提の常時アニメーション

## 6. 実装提案の通し方

1. これはどの Phase の実装か
2. この機能は 3つの判断基準のどれを満たすか
3. 何を壊さない必要があるか
4. 画面のどこに出すか
5. 監査ログは何を残すか
6. テストで何を確認するか
7. docs をどこまで更新するか
