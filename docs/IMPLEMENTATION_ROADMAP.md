# ロボらぼ 実装優先ロードマップ

最終更新日: 2026-03-28

この文書は、ロボらぼの実装優先順位を忘れないための正本メモです。
`docs/GAME_DIRECTION.md` に書かれた運営思想を、具体的な実装順に落としたものとして扱います。

新機能、改善案、UI調整、運用施策を考えるときは、
「今どの Phase を進めているか」
「その実装が Phase の目的に合っているか」
を必ず確認してください。

## 1. この文書の前提

ロボらぼは現在 `チュートリアルフェーズ` にある。

今の探索・強化・進化・ボス・DECOR取得は、
完成したゲーム本編ではなく `基礎体験の提供段階` である。

当面の優先は次の通り:

- 周回快感を上げる
- 型 / 思想の自然発生を作る
- 世界ログ / ランキング / 展示 / 陣営戦で間接競争を育てる
- プレイヤー同士の研究・語り・記録文化を作る

`PvP は入口ではなくゴール` とする。
早期に PvP を主役にしない。

## 2. 判断基準

今後の実装は、必ず次の3条件で判断する。

### 2-1. 周回が気持ちよくなるか

確認観点:

- 毎周回の入力ストレスが減るか
- 前進感が増えるか
- 勝ち体験や報酬感が気持ちよくなるか
- もう1周したくなるか

### 2-2. 競争が自然発生するか

確認観点:

- 他人との比較軸が自然に生まれるか
- ランキング、記録、展示、陣営戦に繋がるか
- プレイヤーごとの差が可視化されるか
- 間接競争を強めるか

### 2-3. プレイヤーの語りが生まれるか

確認観点:

- スクショを貼りたくなるか
- X / Discord で話題にしやすいか
- 自分の育成思想を語れるか
- 他人のロボを見て真似や研究が生まれるか

## 3. 固定制約

今後の実装では、次を壊さないこと。

- 8ターン戦闘を維持する
- `audit.*` 互換を維持する
- `turn_logs` 互換を維持する
- `SYSTEM` 投稿の既存方針を壊さない
- 実装変更時は `docs` 更新と `tests` 更新を必須とする
- 公開は `一気に見せる` のではなく、`管理者確認 -> 段階公開` を優先する

## 4. 実装優先順

## Phase 0: 前進感を見える化する

目的:

- 周回するたびに「前進している」感覚を強くする
- チュートリアル離脱を減らす
- 世界が動いていることを軽く見せ始める

進捗メモ:

- 2026-03-25 初回実装済み
- 進化コアゲージと進化成功・ボス撃破の世界ログ演出を先行反映
- `次の目標` と battle結果の `前進` は重複感が強かったため採用しない
- 進化導線は第2層固定ボス撃破後に初めて表示する方針へ調整

順序:

### 1. 進化コアゲージ

狙い:

- 進化コア入手の運依存感を減らす
- 出撃ごとの前進感を作る
- 進化導線の離脱を減らす

完成条件の例:

- 低確率ドロップは維持
- 別で進化コアゲージを持つ
- 勝利ごとに加算
- 一定値で進化コア保証付与
- ホーム / 進化画面のどちらかで短く進捗表示

### 2. battle結果面の整理

狙い:

- 勝利後に「何を得たか」「何が前進したか」が一目で分かるようにする
- 周回の気持ちよさを上げる

完成条件の例:

- ドロップ / 保証 / ボス報酬の区別が分かる
- 不安を生む文言を減らす
- 進化コアや記録進捗が結果画面で自然に理解できる

### 3. ホームの次目標表示

狙い:

- 「今なにを目指せばいいか」を短く見せる
- ホームで迷わせない

完成条件の例:

- 長文説明を増やしすぎない
- 次の進化コア、次のボス、次の解放などが短く見える
- 初心者にも周回勢にも邪魔にならない

### 4. 進化成功 / ボス撃破の世界ログ強化

狙い:

- 個人の出来事を世界の出来事に変える
- 世界が動いている感じを強める

完成条件の例:

- 進化成功が世界ログで目立つ
- ボス撃破が世界ログで目立つ
- プレイヤー名 / ロボ名 / ボス名が分かる
- X で話題にしやすい

## Phase 1: 理解しやすさを整える

目的:

- 世界観を壊さず、プレイヤーが迷う場所を減らす
- チュートリアル後半の理解コストを下げる

進捗メモ:

- 2026-03-25 初回実装済み
- ホームに `探索先メモ` を追加
- 進化合成画面を `進化状況 / 保証進捗 / 候補件数` が見える形に整理
- ホームの行動導線を `育成 / 管理` で整理
- 実況 / 分析テンプレ用カテゴリを `docs/ANALYSIS_TEMPLATE_CATEGORIES.md` に整理

順序:

### 5. 出撃先ごとの特徴文表示

狙い:

- 「どこへ行くと何が起きやすいか」を短く伝える
- 後の探索傾向差の伏線にする

完成条件の例:

- 各出撃先に短い特徴文がある
- 長文チュートリアル化しない
- 出撃判断の助けになる

### 6. 進化合成画面の改善

狙い:

- 進化導線を分かりやすくする
- コアの価値と用途を自然に理解させる

完成条件の例:

- 進化前 / 進化後が見やすい
- 必要コア数が分かりやすい
- 今どれくらい進化に近いか分かる

### 7. ホーム育成導線整理

狙い:

- ホームを「何を押せばいいか分かる基地」にする
- 目標導線、進化導線、周回導線を邪魔なく整理する

完成条件の例:

- カードが増えすぎない
- 必要な導線だけ見える
- 閉じる / 再表示などの調整が自然

### 8. AI実況分析テンプレ作成用の課題カテゴリ整理

狙い:

- 今後の実況 / 分析 / 運営ネタ化に備える
- どんな型・話題・記録が映えるかを整理する

注意:

- これはプレイヤー向け本線実装ではなく `並行の運営タスク`
- Phase 1 を止める理由にしない

## Phase 2: 型と思想を育てる

目的:

- 周回先の違いからビルド差を自然発生させる
- 「何を目指して育てたか」を可視化する

進捗メモ:

- 2026-03-25 初回実装済み
- 探索場所ごとに軽い育成傾向差を追加
- ホーム / ロボ詳細 / 所有ロボ / 展示で `思想` と `注目能力` を可視化
- `最速 / 耐久 / 命中 / 爆発` の目的別ランキングを追加
- ロボ展示は `最速 / 耐久 / 命中 / 爆発` 並び替えに対応
- `audit.drop` に育成傾向メタデータを残すようにした
- 2026-03-28 第4層を `forge / haze / burst / final` で追加
- 第4層は既存 trait (`heavy / fast / berserk / unstable`) の組み合わせで、耐久 / 命中 / 爆発の勝ち場所を分岐させる
- `records` に第4層初到達を追加し、固定ボス7体体制へ拡張
- 2026-03-28 第5層を `labyrinth / pinnacle / final` で追加
- `records` に第5層初到達を追加し、固定ボス10体体制へ拡張

順序:

### 9. 探索場所ごとの育成傾向差

狙い:

- 探索先の違いで育成方向が変わるようにする
- 努力値のような差の土台を作る

完成条件の例:

- 探索場所ごとに育成の偏りがある
- 一本道の最適解になりすぎない
- 周回先を選ぶ意味が出る

### 10. 型ラベルの可視化

狙い:

- 生まれた差を「語れる形」にする
- 思想を名前で見せる

完成条件の例:

- 耐久型 / 爆発型 / 命中型などが分かる
- プレイヤーが自分のロボを説明しやすい
- 展示やランキングに繋がる

### 11. 目的別ランキング追加

狙い:

- 型 / 思想ごとの間接競争を生む

完成条件の例:

- 最速
- 耐久
- 爆発
- 周回量
- ボス討伐

など、複数の勝ち筋が見える

### 12. ロボ展示強化

狙い:

- プレイヤー同士の研究文化を作る
- 他人の型を見る動機を増やす

完成条件の例:

- 展示が見た目だけでなく思想も伝える
- 並び替えや注目軸が増える
- 真似したくなる導線がある

### 12.5 第4層で型を成立させる

狙い:

- 背水一強をナーフではなく相性分化で崩す
- 耐久 / 命中 / 爆発 / 安定にも明確な勝ち場所を作る
- 第5層の競争文化へ繋ぐ土台を用意する

完成条件の例:

- `layer_4_forge / layer_4_haze / layer_4_burst` の3サブエリアがある
- 各エリアに対応した固定ボスがいる
- 第4層3ボス撃破で `layer_4_final` が解放される
- growth tendency / records / boss reward が第4層まで揃っている
- `layer_4_final` 撃破で第5層が解放される
- 第5層は `layer_5_labyrinth / layer_5_pinnacle / layer_5_final` の構成

## Phase 3: 競争文化を作る

目的:

- 型・思想・周回差を、文化として世界に定着させる
- 擬似対人競争を強くする

進捗メモ:

- 2026-03-26 初回実装済み
- `/world` を追加し、週環境・熱源・陣営戦・研究進捗を1枚で見返せるようにした
- `/records` を追加し、初達成記録 / 今週の記録 / 話題ロボを見返せるようにした
- `/feed?type=weekly` に `陣営戦決着` と `研究解禁` を追加し、週単位の世界変化を公開面でも追えるようにした
- ホームの `今週のランキング`、`世界戦況` の MVP、`記録庫` で `アイコン+小ロボ` と機体画像を併用し、他プレイヤーの存在感を強化した
- `通信` を追加し、`/comms/world` は世界級イベントと全体発言、`/comms/rooms` は公式会議室、`/comms/personal` は個人の成長記録に役割分担した
- ホーム内の `通信` パネルから `世界ログ / 会議室 / 陣営通信 / 個人ログ` をページ内即時タブ切替で見られるようにした
- ホームでは `通信` を `今週のMVP` の直下へ移し、`メニュー` から重複導線の `世界戦況 / 通信 / ランキング` を外した
- 右列は `アクション / 今週の戦況 / 陣営戦 / メニュー / 今週のランキング / 表示調整` の順へ固定した
- ヘッダーは `設定 -> アイコン変更` に整理し、フッターは `通信 / 用語` を外した
- `/guide` に `性格（安定 / 背水 / 爆発）` を追加し、更新履歴 `v0.1.14` を 2026-03-26 内容へ更新した
- VPS 本番は `http://127.0.0.1:8000/healthz` と `https://robolabo.site/healthz` の両方で 200 を確認済み
- 次の有力候補は、週間記録のスナップショット保存と `記録庫` のアーカイブ化

### 実験室寄り道導線メモ

進捗メモ:

- 2026-03-28 実験室レースとエネミーレースを `共通レースエンジン` へ統合
- 2026-03-28 `/admin/release` を追加し、`実験室 / 第4層 / 第5層` を段階的に一般公開できるようにした
- レース本体は `6体固定 / 10区間固定 / 特殊区間 2〜5 抽選` に整理し、見やすさと予想しやすさを優先
- 観戦レースとエネミーレースは同じ watch 画面・同じフレーム生成を使い、差分は `予想 / lab_coin / 倍率` などの周辺仕様だけに寄せた
- 今後の拡張は「障害物を増やすが、1レースで出す数は絞る」方針を維持する

順序:

### 13. 週環境の見える化

狙い:

- 今週何が強いのか
- 何が追い風なのか
- どこが盛り上がっているのか

を見えるようにする

### 14. 陣営の意味の可視化

狙い:

- 陣営をただの所属情報で終わらせない
- 間接競争の大きな軸にする

注意:

- `13. 週環境の見える化` と `14. 陣営の意味の可視化` は実質セットで考える
- 片方だけだと意味が弱くなる

### 15. 記録文化の実装

狙い:

- 世界ログ、展示、ランキングを「文化」にする
- 語りと誇示の場を作る

完成条件の例:

- 初達成記録
- 最速記録
- 今週の記録
- 話題ロボ
- 世界ログと会議室で語りが受け止められる
- 個人ログで前進が見返せる
- 記録の見返し
- ホーム / 世界戦況 / 記録庫で「誰のロボか」が自然に伝わる

が自然に蓄積される

## 5. 今やらないこと

現時点では、次を主戦場にしない。

- 早期 PvP
- 長文チュートリアルの大量追加
- ホームの説明カード増殖
- 一本道の最適解を強める実装
- 直接勝率を売る収益化

## 6. 実装提案の通し方

新しい提案を出すときは、次の順で整理する。

1. これはどの Phase の実装か
2. この機能は 3つの判断基準のどれを満たすか
3. 何を壊さない必要があるか
4. 画面のどこに出すか
5. 監査ログは何を残すか
6. テストで何を確認するか
7. docs をどこまで更新するか
